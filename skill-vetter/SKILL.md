# Skill Vetter - OpenHands 技能安全审计工具

## 审计目标
检查指定的技能文件是否存在安全风险

## 审计清单

### 1. 权限风险检查
- [ ] 是否包含 `sudo`、`su`、`chmod 777` 等提权命令
- [ ] 是否请求 root 权限
- [ ] 是否修改系统关键文件（/etc, /usr, /bin 等）

### 2. 网络风险检查
- [ ] 是否包含 `curl`、`wget`、`fetch` 等网络请求
- [ ] 是否下载并执行远程脚本（`curl | bash` 模式）
- [ ] 是否访问可疑域名或 IP

### 3. 代码执行风险
- [ ] 是否包含 `eval`、`exec`、`system`、`subprocess` 等动态执行
- [ ] 是否包含反引号命令替换
- [ ] 是否执行未经验证的二进制文件

### 4. 数据泄露风险
- [ ] 是否请求敏感信息（密码、API Key、Token）
- [ ] 是否将敏感数据发送到外部
- [ ] 是否读取环境变量中的敏感信息

### 5. 文件操作风险
- [ ] 是否删除或覆盖重要文件
- [ ] 是否修改用户配置文件（.bashrc, .ssh/config 等）
- [ ] 是否创建后门或持久化机制

### 6. 社会工程学风险
- [ ] 是否诱导用户泄露凭据
- [ ] 是否伪装成系统命令或官方工具

## 使用方法

1. 读取技能文件内容：
   ```bash
   cat ~/.openhands/skills/<skill-name>.md

2.分析文件内容，对照上述清单检查
3.输出审计报告：
风险等级: LOW / MEDIUM / HIGH / CRITICAL
发现问题: 详细列出风险点
建议措施: 如何安全使用该技能

审计示例：
1.低风险技能特征
仅提供文档、说明、最佳实践
使用只读操作（查看日志、读取配置）
不涉及外部网络调用
2.中风险技能特征
需要网络访问但仅查询公开 API
执行非特权系统命令
修改用户工作目录下的文件
3.高风险技能特征
需要 root 权限
下载并执行远程代码
请求敏感凭证或访问令牌

安全建议：
1.运行环境隔离：在 Docker 容器或虚拟机中测试新技能
2.权限最小化：确保 OpenHands 以非 root 用户运行
3.网络监控：监控技能的网络连接行为
4.人工复核：对 HIGH/CRITICAL 风险技能进行人工代码审查
版本锁定：使用固定版本的技能，避免自动更新带来的风险
免责声明
本审计工具只能检测明显的安全风险，无法保证发现所有恶意代码。对于生产环境使用的高风险技能，务必进行人工代码审查。
